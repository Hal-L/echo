#### <font color='red'>网络编程三要素</font>

##### IP

```
概念
	每个设备在网络中唯一标识
域名
	其实也是ip,访问域名的时候会先经过DNS域名服务器解析器,将域名解析成IP地址  
指令
	ipconfig:查询本地IP地址
	ping:检查网络是否连通
其他
	127.0.0.1 本地回环地址 和localhost是一个意思
	localhost 本地DNS解析的127.0.0.1的域名
```

##### 端口

```
概念
	每个程序在设备上的唯一标识(传输数据除了要确定发送到哪个设备,还有确定发送到哪个程序)
	
常用端口号
	Tomcat:8080
	web:80
	mysql:3306
	oracle:1521
	qq:4000
	feiq:2425
注意:
	尽量使用1024以上的,1024以下基本上被系统给占用了
解决端口号被占用
	1.netstat -anno 找到端口对应的pid(进程的标记)
	2,在任务管理器中找到对应pid的进程,结束进程
```

##### 协议

```
概念
	用来约定数据的传输规则
分类
	UDP:面向无连接,数据不安全,速度快,不区分客户端和服务器端(发短信,共屏)
	TCP:面向连接(3次握手),数据安全,速度略低,分为客户端和服务器端
	
使用场景
	UDP:对实时性要求很高的音视频通信,如视频会议,视频监控
	用户是无法忍受过高的延迟,但是偶尔丢个包是没有问题
	
	TCP:对实时性要求不高的音视频点播应用.如各大视频网站
	用户是可以忍受延迟,可以进行缓冲,但是不能忍受丢包导致的马赛克
	
	总结:及时性高于可靠性用UDP,可靠性高于及时性用TCP
```



#### Inetaddress类

##### 概念

```
java中专门用来标识网络地址的类,用它创建的对象,包含了主机名和数字类型的ip地址 
```

##### 方法

```
静态方法(获取InetAddress对象)
	static InetAddress getByName(String host)  
	通过传入主机名或者ip地址字符串来获取一个对象
	区别:传入主机名有可能重复,重复了只能获取其中一个,而传入ip,获取就是唯一的
	
	static InetAddress getLocalHost()  
	返回本地主机.如果你仅仅是想获取本地主机,就可以直接使用这个
	
成员方法(获取InetAddress对象里面的信息)
	String getHostAddress() 返回文本显示中的IP地址字符串。 
    String getHostName()  获取此IP地址的主机名。 
```



#### UDP协议

##### 快速入门

```
发送端
	1,创建发送端Socket对象(相当于创建码头)
	2,创建数据并打包(相当于创建集装箱)
	3,发送数据(相当于发货,将货物发出)
	4,释放资源(相当于关闭码头)

接收端
	1,创建接收端Socket对象(相当于创建码头)
	2,接收数据(相当于创建集装箱接货)
	3,解析数据(相当于打开集装箱)
	4,释放资源(相当于关闭码头)
```

##### 通讯方式

```
单播:一对一个

组播:一对一组
	发送端
		发送数据的时候,填写的是组播地址(224.0.1.0-239.255.255.255)
	接收端
		使用MulticastSocket创建接收端Socket对象
		在接收数据之前,把当前的计算机绑定到一个组播地址

广播:一对全部
	发送端
		发送数据的时候,填写的是组播地址(255.255.255.255)
```



#### <font color='red'>TCP协议</font>

##### 快速入门

```
客户端
	1.创建发送端Socket对象
	2,获取输出流对象
	3,发送数据
	4,释放资源
服务器端
	1.创建接收端Socket对象
	2,获取输入流对象
	3,接收数据
	4,释放资源
```

##### 三次握手

```
目的:双方确定自己和对方的发送和接收都是正常的

Client:嗨 能听到吗
Server:我听到了,你能听到我吗
Client:好的,我们都能听到,可以开始通话了

第一次握手:Client什么都不能确定;Server可以确定对方发送正常
第二次握手:Client确定:自己发送,接收正常的,对方发送,接收都是正常;Server可以确定自己接收正常,对方发送正常
第三次握手:Client确定:自己发送,接收正常的,对方发送,接收都是正常;Server可以确定自己发送,接收正常,对方发送,接收正常
```

##### 四次挥手

```
Client:我所有的东西都说完了
Server:我已经全部听到了,但是等等我,我还没说完
Server:好了,我已经说完了
Client:好的,我们通讯就结束了
```
